# プロジェクト概要
対話形式で実験レポートの作成を支援するWebアプリケーションを開発します。
単にAIがレポートを代筆するのではなく、「AIが実験データや図表を提示し、ユーザーに考察をファシリテートする（問いかける）」ことで、ユーザー自身に考えさせる学習支援システムを目指します。

# 実行環境と制約（重要）
- **環境**: WSL (Windows Subsystem for Linux) 上のUbuntu環境。
- **ファイルアクセス**: Pythonスクリプト、Streamlitサーバー、および `subprocess` で呼び出すコマンド（pdflatex, gnuplot）はすべて同Linux環境内で完結して実行されます。パスの指定はLinuxの絶対パスまたは相対パスを使用し、ファイルの読み書きは必ず `encoding="utf-8"` を明記してください。

# 技術スタック
- **フロントエンド・UI**: Streamlit (Python)
- **バックエンド**: Python 3.x
- **AI API**: Gemini 3 Flash API (Google GenAI SDKを使用)
- **テキスト抽出**: ndlocr-lite (PDFからの日本語テキスト抽出)
- **データ処理**: pandas, openpyxl (Excelデータの読み込みと整形)

# システムの全体構成と処理フロー
以下の順序で処理を行うシステムを構築してください。

1. **初期化とファイル読み込み**
   - ユーザーが対象ディレクトリを指定。
   - ディレクトリ内にある「実験データのExcel」「指導書のPDF」「レポートテンプレート(TeX)」を読み込む。
2. **データのパースと前処理**
   - Excel: pandasで読み込み、AIが理解しやすいMarkdownの表に変換。
   - PDF: ページごとに画像化(PNG)し、同時に `ndlocr-lite` でテキストを抽出。
   - TeXテンプレート: 個人情報保護のため、表紙やプリアンブル部分と、AIが編集する本文部分を特定の目印（例: `% --- AI_TARGET_START ---`）で文字列分割し、本文のみを変数に保持する。
3. **Gemini APIによるルール抽出**
   - 「PDFの画像全ページ」「ndlocr-liteの抽出テキスト」「Excelデータ」をGemini 3 Flashに送信。
   - 指示: 「テキストデータを正として扱い、画像からグラフ作成のルール（軸の設定など）やレポートのフォーマットを読み取ってください。」
4. **チャットUIによるファシリテーション（最重要機能）**
   - StreamlitのチャットUI上で、AIがユーザーに問いかける。
   - **AIの役割（システムプロンプト）**: 「あなたは高専の電気電子工学の実験指導教員です。絶対に答えを先に言わず、データや作成した図を示しながら『このオシロスコープの波形や測定結果から、何が読み取れますか？』等と問いかけ、ユーザーに考察を促してください。間違っていたらヒントを出してください。」
5. **コード生成とコンパイル**
   - 対話が完了したら、AIにgnuplotの描画スクリプトと、TeXの本文コードを生成させる（※自然言語を含めずコードのみを出力するよう厳格に指示、またはStructured Outputsを使用）。
   - Pythonの `subprocess` でgnuplotを実行し画像を生成。
   - 分割しておいたTeXの表紙部分とAIが生成した本文を再結合して `pdflatex` でコンパイルする。
6. **レビューと完了**
   - Streamlit上で進捗プログレスバーを100%にし、PDFのプレビューやダウンロードボタンを表示する。
   - AIからの最終レビューをチャットに出力する。

# 実装において特に注意すべき難所
以下の点はバグの温床になりやすいため、重点的にエラーハンドリングを行ってください。
- **Streamlitの状態管理**: 再描画によってチャット履歴や処理フェーズがリセットされないよう、`st.session_state` を適切に設計してください。
- **外部プロセスのエラーハンドリング**: `subprocess` でgnuplotやtexを実行する際、構文エラーが発生し得ます。エラー時は標準エラー出力を取得し、適切にハンドリング（またはAIに再修正を要求）してください。
- **個人情報の分離**: TeXテンプレートの分割と結合のロジックは、絶対にAPIへの通信前に実行し、個人情報（氏名や学籍番号など）が外部APIに送信されない設計を厳守してください。

# 追加の技術指示
- **Git管理**: `git init` から開始し、フェーズごとに差分が明確になるように管理すること。また、APIキーなど公開してはいけないものは絶対に.gitignoreに記載すること。
- **ログ出力**: 開発中のデバッグを容易にするため、各フェーズの処理結果（抽出されたテキストや生成された中間ファイルなど）をターミナルやStreamlit上にログとして出力する機能を設けること。
